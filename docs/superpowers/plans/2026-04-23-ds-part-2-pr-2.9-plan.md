# Design System SP1 Part 2 — PR 2.9 Implementation Plan (Breadcrumb + DataTable)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the last two custom (non-Radix) compounds — `Breadcrumb` and `DataTable` — and adopt `DataTable` on the headless XP-breakdown table. `Breadcrumb` ships without adoption because Settings has no navigable hierarchy in today's IA.

**Architecture:** Both are thin, dependency-free wrappers. `Breadcrumb` renders an `<ol>` of `<Link>`-or-`<span>` items separated by a Lucide `ChevronRight`; the last item is always a plain `<span>` (current page, no link). `DataTable` renders a styled `<table>` with an optional header row and a per-column `render(row)` function plus `align` and `className` overrides; `getRowKey` provides stable `<tr>` keys. No sorting/paging — consumer's problem per spec §4.

**Tech Stack:** TS 5, React 19, `next/link`, `lucide-react` (ChevronRight), Vitest 4 + RTL (jsdom pragma), `cn()`.

**Scope references:**
- Parent spec: `docs/superpowers/specs/2026-04-21-design-system-foundation-part-2-design.md` §4 (compound table — Breadcrumb Custom, DataTable Custom thin wrapper), §10 PR 2.9
- Previous PRs: 2.6 OPEN (#6), 2.7 OPEN (#7), 2.8 OPEN (#8). Main tip at branch-off: `83a3da9`.
- Parallel: independent of #6/#7/#8 file-wise, but 2.7 and 2.8 both create `src/components/ui/compound/` with a barrel; this PR also extends that barrel — expect one trivial `compound/index.ts` conflict against whichever of {#7, #8} merges first.

**Deviations from parent spec (explicit so reviewer can push back):**
1. **Breadcrumb — no adoption this PR** — spec §10 PR 2.9 says "Breadcrumb on Settings sub-pages if they exist". Settings has 3 sub-pages (`/settings/macros`, `/settings/plates`, `/settings/export`) but crucially has **no Settings index page, no shared `layout.tsx`, and no cross-navigation between the three**. Each sub-page is reached only from the bottom-nav `Nastavení` tab (which hard-codes `/settings/plates`). Surfacing a breadcrumb "Nastavení › Makra" would imply a parent page that doesn't exist and a sibling-nav that doesn't exist either. Ship Breadcrumb as available primitive; adoption lands if Settings grows an index or SP2 information-architecture adds real hierarchy.
2. **DataTable — adopted only on XpBreakdown, NOT MeasurementGrid** — spec §10 PR 2.9 says "DataTable on measurements history". `MeasurementGrid` rows are stateful React components (`MeasurementRow`) with inline-editable cells, commit handlers, current-week highlight, and an expandable note row that spans 8 columns as a second `<tr>`. A generic thin wrapper can't express spanned rows + cell-level state without an escape hatch that would negate the simplification. `XpBreakdown` is the clean fit: headless 4-column read-only read from `byEventTotal` — migrates cleanly to DataTable with no feature loss.
3. **DataTable API wider than spec literal** — spec §4 says `<DataTable headers data />` + `renderCell`. We ship `<DataTable columns={[{key, header?, align, render}]} data={} getRowKey={} />`. Rationale: XpBreakdown's columns have different alignments (left / right / right / right + small-text-xs on the percent column) and no `<thead>` at all. A single flat `renderCell` map couldn't express column-level alignment or header-presence. Columns-array keeps consumer ergonomics (one object per column).

**Out of scope:**
- Breadcrumb adoption on Settings (no hierarchy to surface)
- DataTable adoption on MeasurementGrid (too domain-heavy for thin wrapper)
- Sorting / pagination (consumer's problem per spec §4)
- Storybook / markdown docs (PR 2.11)

Create `src/components/ui/compound/Breadcrumb.tsx`:

```tsx
import Link from 'next/link'
import { ChevronRight } from 'lucide-react'
import { cn } from '@/components/ui/utils/cn'

type BreadcrumbItem = { label: string; href?: string }

type Props = {
  items: BreadcrumbItem[]
  className?: string
}

export function Breadcrumb({ items, className }: Props) {
  return (
    <nav aria-label="Breadcrumb" className={cn('text-sm', className)}>
      <ol className="text-muted flex flex-wrap items-center gap-1.5">
        {items.map((item, i) => {
          const isLast = i === items.length - 1
          return (
            <li key={`${item.label}-${i}`} className="flex items-center gap-1.5">
              {isLast ? (
                <span aria-current="page" className="text-foreground font-medium">
                  {item.label}
                </span>
              ) : item.href ? (
                <Link href={item.href} className="hover:text-foreground transition-colors">
                  {item.label}
                </Link>
              ) : (
                <span>{item.label}</span>
              )}
              {!isLast && (
                <ChevronRight
                  size={14}
                  data-breadcrumb-separator
                  className="text-muted/60"
                  aria-hidden
                />
              )}
            </li>
          )
        })}
      </ol>
    </nav>
  )
}
```

**API shape:**

```tsx
type Align = 'left' | 'right' | 'center'

type Column<T> = {
  key: string
  header?: string
  align?: Align
  render: (row: T) => ReactNode
  headerClassName?: string
  cellClassName?: string
}

type Props<T> = {
  columns: Column<T>[]
  data: T[]
  getRowKey: (row: T, idx: number) => string | number
  className?: string
  empty?: ReactNode
}
```

Behaviour:
- If no column has `header`, `<thead>` is omitted (headless table).
- `align` maps to `text-left | text-right | text-center` on both header and cell.
- Empty-state: if `data.length === 0` and `empty` prop is set, render `empty` instead of `<tbody>`; fall back to `<tbody/>` if `empty` is omitted.

Create `src/components/ui/compound/DataTable.tsx`:

```tsx
import type { ReactNode } from 'react'
import { cn } from '@/components/ui/utils/cn'

type Align = 'left' | 'right' | 'center'

export type DataTableColumn<T> = {
  key: string
  header?: string
  align?: Align
  render: (row: T) => ReactNode
  headerClassName?: string
  cellClassName?: string
}

type Props<T> = {
  columns: DataTableColumn<T>[]
  data: T[]
  getRowKey: (row: T, idx: number) => string | number
  className?: string
  empty?: ReactNode
}

const ALIGN: Record<Align, string> = {
  left: 'text-left',
  right: 'text-right',
  center: 'text-center',
}

export function DataTable<T>({ columns, data, getRowKey, className, empty }: Props<T>) {
  const hasHeaders = columns.some((c) => c.header !== undefined)
  const showEmpty = data.length === 0 && empty !== undefined
  return (
    <table className={cn('w-full border-collapse text-sm', className)}>
      {hasHeaders && (
        <thead>
          <tr className="border-border border-b">
            {columns.map((c) => (
              <th
                key={c.key}
                className={cn(
                  'text-muted px-1.5 py-2 text-[11px] font-medium',
                  ALIGN[c.align ?? 'left'],
                  c.headerClassName
                )}
              >
                {c.header}
              </th>
            ))}
          </tr>
        </thead>
      )}
      {showEmpty ? (
        <caption className="text-muted py-4 text-sm caption-bottom">{empty}</caption>
      ) : (
        <tbody>
          {data.map((row, idx) => (
            <tr key={getRowKey(row, idx)} className="border-border border-b last:border-0">
              {columns.map((c) => (
                <td
                  key={c.key}
                  className={cn(
                    'px-1.5 py-2',
                    ALIGN[c.align ?? 'left'],
                    c.cellClassName
                  )}
                >
                  {c.render(row)}
                </td>
              ))}
            </tr>
          ))}
        </tbody>
      )}
    </table>
  )
}
```

Note on empty-slot: the test asserts `container.querySelector('tbody')` is null. Using `<caption>` keeps semantics (screen readers announce it) and leaves tbody absent. If the `<caption>` approach causes layout issues in real use later, swap to `<tbody><tr><td colSpan={n}>empty</td></tr></tbody>`; for now this is simplest and passes the test contract.

```tsx
import { DataTable, type DataTableColumn } from '@/components/ui'
import type { XpHistory } from '@/lib/queries/xp-history'

const LABELS: Record<string, string> = {
  session_complete: 'Dokončené tréninky',
  set_logged: 'Série zalogované',
  measurement_added: 'Měření',
  photo_uploaded: 'Fotky',
  nutrition_logged: 'Výživa',
  pr_achieved: 'PR překonán',
  streak_day: 'Streak',
}

type Row = { event: string; xp: number; count: number }

type Props = { byEventTotal: XpHistory['byEventTotal']; total: number }

export function XpBreakdown({ byEventTotal, total }: Props) {
  const rows: Row[] = Object.entries(byEventTotal)
    .map(([event, v]) => ({ event, xp: v!.xp, count: v!.count }))
    .sort((a, b) => b.xp - a.xp)

  const columns: DataTableColumn<Row>[] = [
    {
      key: 'event',
      render: (r) => LABELS[r.event] ?? r.event,
      cellClassName: 'text-foreground py-2',
    },
    {
      key: 'count',
      align: 'right',
      render: (r) => `${r.count}×`,
      cellClassName: 'text-muted py-2',
    },
    {
      key: 'xp',
      align: 'right',
      render: (r) => `${r.xp.toLocaleString('cs-CZ')} XP`,
      cellClassName: 'text-foreground py-2 font-semibold',
    },
    {
      key: 'percent',
      align: 'right',
      render: (r) => `${total > 0 ? Math.round((r.xp / total) * 100) : 0}%`,
      cellClassName: 'text-muted py-2 pl-3 text-xs',
    },
  ]

  return (
    <div className="border-border bg-surface rounded-lg border p-4">
      <h2 className="text-foreground mb-3 text-sm font-semibold">Rozpis podle aktivity</h2>
      <DataTable
        columns={columns}
        data={rows}
        getRowKey={(r) => r.event}
        empty={<span>Zatím žádná aktivita.</span>}
      />
    </div>
  )
}
```
