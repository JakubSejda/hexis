# M1 — Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implementovat NextAuth v5 auth pipeline (Credentials + Google OAuth) s JWT sessions, user bootstrap CLI, security middleware, rate limiting a ownership pattern. Po M1 bude možné se přihlásit jako Jakub a vstoupit do chráněných route.

**Architecture:** NextAuth v5 (Auth.js) s Drizzle adapter, JWT session v httpOnly cookie, argon2id password hashing. Credentials provider jako primární metoda, Google OAuth jako alternativa (vyžaduje Google Cloud Console setup). Middleware pokrývá security headers na všech routách + auth gate na `(app)/*`. Rate limit na auth endpoints jako in-memory sliding window (Fáze 2+ přesun do Redis). Helper `requireOwnership` jako konzistentní pattern napříč budoucími API endpoints.

**Tech Stack:** NextAuth v5 (next-auth@beta), @auth/drizzle-adapter, @node-rs/argon2 (native, fast), zod (validation already installed), bcrypt-pbkdf (fallback), react-hook-form (login form).

**Next.js 16 warning:** AGENTS.md v repu varuje že Next.js 16 má breaking changes. NextAuth v5 je primárně psaný pro Next.js 14/15. Potenciální issues: middleware syntax, route handler signature, cookie API. Pokud narazíš na runtime error, prvním krokem je **přečíst `node_modules/next/dist/docs/` relevantní sekci**, ne guessnout opravu.

```typescript
// src/lib/password.ts
import { hash, verify } from '@node-rs/argon2'

// argon2id s OWASP 2024 doporučenými parametry pro web apps
// - memoryCost: 19 MB (19456 KB)
// - timeCost: 2 iterations
// - parallelism: 1
const ARGON2_OPTIONS = {
  memoryCost: 19456,
  timeCost: 2,
  parallelism: 1,
} as const

export async function hashPassword(plain: string): Promise<string> {
  return hash(plain, ARGON2_OPTIONS)
}

export async function verifyPassword(storedHash: string, plain: string): Promise<boolean> {
  try {
    return await verify(storedHash, plain)
  } catch {
    // Invalid hash format → reject
    return false
  }
}
```

```typescript
// src/lib/auth.ts
import NextAuth from 'next-auth'
import Credentials from 'next-auth/providers/credentials'
import Google from 'next-auth/providers/google'
import { DrizzleAdapter } from '@auth/drizzle-adapter'
import { eq } from 'drizzle-orm'
import { db } from '@/db/client'
import { users } from '@/db/schema'
import { env } from '@/lib/env'
import { verifyPassword } from '@/lib/password'
import { loginSchema } from '@/lib/validation'

export const { handlers, auth, signIn, signOut } = NextAuth({
  adapter: DrizzleAdapter(db),
  session: {
    strategy: 'jwt',
    maxAge: 30 * 24 * 60 * 60, // 30 dní
  },
  secret: env.AUTH_SECRET,
  pages: {
    signIn: '/login',
  },
  providers: [
    Credentials({
      name: 'Credentials',
      credentials: {
        email: { label: 'Email', type: 'email' },
        password: { label: 'Password', type: 'password' },
      },
      async authorize(rawCredentials) {
        const parsed = loginSchema.safeParse(rawCredentials)
        if (!parsed.success) return null

        const { email, password } = parsed.data
        const [user] = await db
          .select()
          .from(users)
          .where(eq(users.email, email))
          .limit(1)

        if (!user?.passwordHash) return null

        const ok = await verifyPassword(user.passwordHash, password)
        if (!ok) return null

        return {
          id: user.id,
          email: user.email,
          name: user.name,
          level: user.level,
        }
      },
    }),
    ...(env.GOOGLE_CLIENT_ID && env.GOOGLE_CLIENT_SECRET
      ? [
          Google({
            clientId: env.GOOGLE_CLIENT_ID,
            clientSecret: env.GOOGLE_CLIENT_SECRET,
            allowDangerousEmailAccountLinking: true, // email = identita
          }),
        ]
      : []),
  ],
  callbacks: {
    async jwt({ token, user }) {
      if (user) {
        token.userId = user.id as string
        token.email = user.email!
        token.level = (user as { level?: number }).level
      }
      return token
    },
    async session({ session, token }) {
      if (session.user) {
        session.user.id = token.userId
        session.user.email = token.email
        session.user.level = token.level
      }
      return session
    },
  },
})
```

**Pozn. k Google OAuth:** Providers array podmínečně zahrnuje Google jen pokud jsou env vars vyplněné. V M1 Task 15 (Google setup) je user action — dokud `.env.local` nemá GOOGLE_CLIENT_ID/SECRET, Google button se nezobrazí v UI.

```typescript
// src/db/bootstrap.ts
import * as dotenv from 'dotenv'
dotenv.config({ path: '.env.local' })
dotenv.config({ path: '.env' })

import { drizzle } from 'drizzle-orm/mysql2'
import { eq } from 'drizzle-orm'
import mysql from 'mysql2/promise'
import * as readline from 'node:readline/promises'
import * as schema from './schema'
import { users, plans } from './schema'
import { hashPassword } from '../lib/password'
import { newUlid } from '../lib/ulid'
import { emailSchema, passwordSchema } from '../lib/validation'

async function prompt(rl: readline.Interface, q: string, hidden = false): Promise<string> {
  if (!hidden) return (await rl.question(q)).trim()

  // Hidden input (password) — node-readline native hidden input není, použijeme trick:
  process.stdout.write(q)
  return new Promise((resolve) => {
    const stdin = process.openStdin()
    let input = ''
    const onData = (char: Buffer) => {
      const c = char.toString()
      if (c === '\n' || c === '\r' || c === '\u0004') {
        stdin.removeListener('data', onData)
        process.stdout.write('\n')
        resolve(input)
      } else if (c === '\u0003') {
        process.exit(130)
      } else {
        input += c
      }
    }
    stdin.on('data', onData)
  })
}

async function main() {
  const url = process.env.DATABASE_URL
  if (!url) throw new Error('DATABASE_URL required')

  const connection = await mysql.createConnection(url)
  const db = drizzle(connection, { schema, mode: 'default' })

  const rl = readline.createInterface({ input: process.stdin, output: process.stdout })

  try {
    console.log('Hexis user bootstrap\n')

    const emailRaw = await prompt(rl, 'Email: ')
    const email = emailSchema.parse(emailRaw)

    const [existing] = await db.select().from(users).where(eq(users.email, email)).limit(1)
    if (existing) {
      console.log(`\nUser s emailem ${email} už existuje (id: ${existing.id}). Konec.`)
      return
    }

    const name = (await prompt(rl, 'Jméno (optional): ')) || null

    const password = await prompt(rl, 'Heslo (8+ znaků, ≥1 velké, ≥1 číslo): ', true)
    passwordSchema.parse(password)

    const passwordHash = await hashPassword(password)
    const userId = newUlid()

    await db.insert(users).values({
      id: userId,
      email,
      passwordHash,
      name,
      level: 1,
    })

    console.log(`\n✓ User vytvořen (id: ${userId})`)

    // Re-assign placeholder seeded plans na tohoto usera
    const PLACEHOLDER = '00000000000000000000000001'
    const result = await db
      .update(plans)
      .set({ userId })
      .where(eq(plans.userId, PLACEHOLDER))

    console.log(`✓ Plány (UA/UB/LA/LB) re-assigned z placeholder na ${userId}`)
  } finally {
    rl.close()
    await connection.end()
  }
}

main().catch((err) => {
  console.error('\nBootstrap failed:', err.message ?? err)
  process.exit(1)
})
```

```typescript
// src/lib/auth-helpers.ts
import { auth } from '@/lib/auth'

/**
 * Vrací aktuálního přihlášeného usera, nebo null.
 * Pro API routy preferuj `requireSessionUser` který rovnou throwne.
 */
export async function getSessionUser() {
  const session = await auth()
  return session?.user ?? null
}

/**
 * Vrací sessionUser, nebo vyhodí 401 Response.
 * Použití v route handlers:
 *   const user = await requireSessionUser()
 *   if (user instanceof Response) return user
 */
export async function requireSessionUser() {
  const user = await getSessionUser()
  if (!user) {
    return new Response(JSON.stringify({ error: 'Unauthorized' }), {
      status: 401,
      headers: { 'content-type': 'application/json' },
    })
  }
  return user
}

/**
 * Ověří že row patří aktuálnímu uživateli.
 * Vrací row pokud OK, jinak 404 Response (nikdy 403 — neleakuje info).
 *
 * Použití:
 *   const row = await requireOwnership(
 *     db.query.sessions.findFirst({ where: eq(sessions.id, id) }),
 *     user.id
 *   )
 *   if (row instanceof Response) return row
 */
export async function requireOwnership<T extends { userId: string }>(
  rowPromise: Promise<T | undefined>,
  userId: string
): Promise<T | Response> {
  const row = await rowPromise
  if (!row || row.userId !== userId) {
    return new Response(JSON.stringify({ error: 'Not found' }), {
      status: 404,
      headers: { 'content-type': 'application/json' },
    })
  }
  return row
}
```

Poznámka k `instanceof Response` pattern: NextAuth v5 API routes vracejí buď data, nebo Response. Pattern umožňuje early-return z route handler bez throwing.

```typescript
// src/lib/ratelimit.ts
type RateLimitOptions = {
  limit: number
  windowMs: number
}

type CheckResult =
  | { allowed: true; remaining: number }
  | { allowed: false; retryAfterMs: number }

/**
 * In-memory sliding-window rate limiter.
 * Vhodné pro Fázi 1 single-instance. Fáze 2+ přesun do Redis.
 */
export function createRateLimiter(opts: RateLimitOptions) {
  const buckets = new Map<string, number[]>()

  function check(key: string): CheckResult {
    const now = Date.now()
    const cutoff = now - opts.windowMs

    const bucket = buckets.get(key) ?? []
    // Vyhodit starší timestampy
    const fresh = bucket.filter((t) => t > cutoff)

    if (fresh.length >= opts.limit) {
      const oldest = fresh[0]!
      return {
        allowed: false,
        retryAfterMs: oldest + opts.windowMs - now,
      }
    }

    fresh.push(now)
    buckets.set(key, fresh)
    return { allowed: true, remaining: opts.limit - fresh.length }
  }

  function reset(key: string) {
    buckets.delete(key)
  }

  return { check, reset }
}

/**
 * Preset limiters pro různé endpoints.
 * Fáze 1 single instance → state se ztratí při restartu (OK pro auth).
 */
export const authLoginLimiter = createRateLimiter({
  limit: 5,
  windowMs: 15 * 60_000, // 15 minut
})
```

- [ ] **Step 1: Middleware implementation**

```typescript
// src/middleware.ts
import { NextResponse, type NextRequest } from 'next/server'
import { authLoginLimiter } from '@/lib/ratelimit'

export function middleware(req: NextRequest) {
  // 1) Rate limit pro credentials callback
  if (req.nextUrl.pathname === '/api/auth/callback/credentials' && req.method === 'POST') {
    const ip =
      req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ??
      req.headers.get('x-real-ip') ??
      'unknown'

    const check = authLoginLimiter.check(ip)
    if (!check.allowed) {
      return new NextResponse(
        JSON.stringify({ error: 'Too many attempts. Try again later.' }),
        {
          status: 429,
          headers: {
            'content-type': 'application/json',
            'retry-after': Math.ceil(check.retryAfterMs / 1000).toString(),
          },
        }
      )
    }
  }

  const res = NextResponse.next()

  // 2) Security headers pro všechny response
  res.headers.set('X-Content-Type-Options', 'nosniff')
  res.headers.set('X-Frame-Options', 'DENY')
  res.headers.set('Referrer-Policy', 'strict-origin-when-cross-origin')
  res.headers.set('Permissions-Policy', 'geolocation=(), microphone=(), camera=()')

  // HSTS jen v produkci (lokální dev přes HTTP)
  if (process.env.NODE_ENV === 'production') {
    res.headers.set('Strict-Transport-Security', 'max-age=31536000; includeSubDomains')
  }

  // CSP — zatím permisivní pro dev; tightnout v Fázi 2
  res.headers.set(
    'Content-Security-Policy',
    [
      "default-src 'self'",
      "img-src 'self' data: blob:",
      "script-src 'self' 'unsafe-inline' 'unsafe-eval'", // Next.js dev vyžaduje
      "style-src 'self' 'unsafe-inline'",
      "connect-src 'self'",
      "font-src 'self' data:",
      "object-src 'none'",
      "base-uri 'self'",
    ].join('; ')
  )

  return res
}

export const config = {
  // Middleware běží na všech routách kromě statických assets
  matcher: ['/((?!_next/static|_next/image|favicon.ico|.*\\.svg).*)'],
}
```

**Poznámka k CSP:** V produkci je potřeba tightnout (odstranit `unsafe-inline` a `unsafe-eval` ze `script-src`). Next.js má nonce-based CSP helper — v Fázi 2 nastavit správně. Pro M1 dev prioritizujeme "works" nad "perfect CSP".
