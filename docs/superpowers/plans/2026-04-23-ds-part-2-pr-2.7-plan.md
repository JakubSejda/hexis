# Design System SP1 Part 2 — PR 2.7 Implementation Plan (Dialog + Tooltip)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship first two compound primitives — `Dialog` (re-shaped single-export wrapping Radix Dialog) and `Tooltip` (re-shaped single-export wrapping Radix Tooltip) — add `description` prop to `BottomSheet` for Dialog API alignment, and adopt Dialog for the 2 existing `window.confirm` calls + Tooltip on StagnationBadge.

**Architecture:** Compound components live under new `src/components/ui/compound/` directory and export via the top-level barrel `@/components/ui`. Dialog uses Radix `@radix-ui/react-dialog` and renames the primary export to avoid collision with Radix's own `Dialog.*` subcomponent names. Tooltip uses `@radix-ui/react-tooltip` with `delayDuration={0}` and relies on focus (not hover) as the mobile fallback — consumers wrap focusable elements. Both ship with tests that exercise open/close behavior, not just smoke.

**Tech Stack:** TS 5, React 19, `@radix-ui/react-dialog` 1.1, `@radix-ui/react-tooltip` 1.2 (both already installed), Vitest 4 + RTL (jsdom pragma), `user-event` for interaction, `cn()`.

**Scope references:**
- Parent spec: `docs/superpowers/specs/2026-04-21-design-system-foundation-part-2-design.md` §4 (compound primitives table), §5 (BottomSheet row), §8 (interaction tests), §10 PR 2.7
- Previous PR: 2.6 (open — PR #6). Main tip at branch-off: `83a3da9`.
- Starts in parallel with PR #6 being in review; Dialog/Tooltip are independent of HeroBanner/ProgressBar/Sparkline changes.

**Deviations from parent spec (explicit so reviewer can push back):**
1. **BottomSheet NOT replaced by Dialog** — spec §10 PR 2.7 says "Adopt Dialog everywhere current BottomSheet is used as modal + confirm dialogs". All 6 existing BottomSheet callsites (`DailyModal`, `UploadSheet`, `ExercisePicker`, `PlateCalculatorSheet`, `EditSetSheet`, `ExerciseStepper`'s indirect use via EditSetSheet) are phone-native slide-up forms, not confirm-style centered modals — replacing them would be a UX regression on mobile. We interpret "used as modal" narrowly: the two `window.confirm` calls (`EditSetSheet.tsx:29`, `ExerciseStepper.tsx:42`) are the actual confirm-style sites. Lightbox's 2-stage inline delete confirm is idiomatic and stays.
2. **Tooltip touch fallback = focus-only** — spec §4 says "tap-delegation handled internally via Radix `delayDuration={0}` + touch fallback". We interpret "touch fallback" as: trigger children must be focusable (`<button>`, `<a>`, or `tabIndex=0`); on touch, tap → focus → tooltip opens. This is how Radix's combined focus/hover listener works with `delayDuration={0}`. No extra Popover wrapper or touch-emulation layer. StagnationBadge (currently a `<span>` with native `title=`) gets `tabIndex={0}` to remain reachable on touch.
3. **No `ConfirmDialog` helper** — YAGNI. 2 confirm sites are small; each gets inline `<Dialog>` with local `useState<boolean>` for open. A helper can land if a 4th site pops up.
4. **Docs deferred** — per PR 2.11 scope; this PR doesn't write markdown docs.

**Out of scope:**
- BottomSheet→Dialog migration of the 6 bottom-sheet form callsites
- Lightbox's 2-stage inline confirm → Dialog
- Storybook / markdown docs (PR 2.11)
- Tooltip adoption on chart axis hints (deferred; no obvious site today)

```tsx
'use client'
import * as DialogPrimitive from '@radix-ui/react-dialog'
import type { ReactNode } from 'react'

type DialogProps = {
  open: boolean
  onOpenChange: (open: boolean) => void
  title: string
  description?: string
  children?: ReactNode
  dismissible?: boolean
}

export function Dialog({
  open,
  onOpenChange,
  title,
  description,
  children,
  dismissible = true,
}: DialogProps) {
  const blockDismiss = !dismissible
  return (
    <DialogPrimitive.Root open={open} onOpenChange={onOpenChange}>
      <DialogPrimitive.Portal>
        <DialogPrimitive.Overlay className="data-[state=open]:animate-in data-[state=open]:fade-in-0 fixed inset-0 z-40 bg-black/50" />
        <DialogPrimitive.Content
          className="border-border bg-surface text-foreground fixed top-1/2 left-1/2 z-50 w-[min(92vw,420px)] -translate-x-1/2 -translate-y-1/2 rounded-xl border p-4 shadow-xl focus:outline-none"
          onEscapeKeyDown={(e) => {
            if (blockDismiss) e.preventDefault()
          }}
          onPointerDownOutside={(e) => {
            if (blockDismiss) e.preventDefault()
          }}
        >
          <DialogPrimitive.Title className="text-base font-semibold">
            {title}
          </DialogPrimitive.Title>
          {description && (
            <DialogPrimitive.Description className="text-muted mt-1 text-sm">
              {description}
            </DialogPrimitive.Description>
          )}
          {children && <div className="mt-4">{children}</div>}
        </DialogPrimitive.Content>
      </DialogPrimitive.Portal>
    </DialogPrimitive.Root>
  )
}

// Escape hatches for consumers that need raw Radix composition
export const DialogRoot = DialogPrimitive.Root
export const DialogTrigger = DialogPrimitive.Trigger
export const DialogPortal = DialogPrimitive.Portal
export const DialogOverlay = DialogPrimitive.Overlay
export const DialogContent = DialogPrimitive.Content
export const DialogTitle = DialogPrimitive.Title
export const DialogDescription = DialogPrimitive.Description
export const DialogClose = DialogPrimitive.Close
```

```tsx
'use client'
import * as TooltipPrimitive from '@radix-ui/react-tooltip'
import type { ReactElement } from 'react'

type Side = 'top' | 'right' | 'bottom' | 'left'

type TooltipProps = {
  content: string
  side?: Side
  children: ReactElement
}

export function Tooltip({ content, side = 'top', children }: TooltipProps) {
  return (
    <TooltipPrimitive.Provider delayDuration={0}>
      <TooltipPrimitive.Root>
        <TooltipPrimitive.Trigger asChild>{children}</TooltipPrimitive.Trigger>
        <TooltipPrimitive.Portal>
          <TooltipPrimitive.Content
            side={side}
            sideOffset={6}
            className="border-border bg-surface-raised text-foreground z-50 max-w-[240px] rounded-md border px-2 py-1 text-xs shadow-lg"
          >
            {content}
            <TooltipPrimitive.Arrow className="fill-surface-raised" />
          </TooltipPrimitive.Content>
        </TooltipPrimitive.Portal>
      </TooltipPrimitive.Root>
    </TooltipPrimitive.Provider>
  )
}

export const TooltipProvider = TooltipPrimitive.Provider
export const TooltipRoot = TooltipPrimitive.Root
export const TooltipTrigger = TooltipPrimitive.Trigger
export const TooltipContent = TooltipPrimitive.Content
```
