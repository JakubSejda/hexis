# Design System SP1 Part 2 — PR 2.11 Implementation Plan (Docs + README)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Write markdown reference docs for every design-system component per spec §9, create a `docs/design-system/README.md` index, and cross-link from the top-level `README.md`. This is the final PR of Part 2.

**Architecture:** Pure markdown. No code changes. 32 component docs + 1 index.

**Scope references:** Spec §9 (markdown template), §10 PR 2.11.

**Dependency:** This PR is forward-looking — it documents components that land in PRs #6 (HeroBanner, ProgressBar variant=xp, Sparkline tone), #7 (Dialog, Tooltip, BottomSheet description), #8 (Tabs, Menu, Accordion), #9 (Breadcrumb, DataTable), and #10 (no new components, but icon sweep references Lucide patterns). **This PR must merge LAST, after #6–#10 are all merged**, otherwise docs describe APIs that don't exist on main yet.

**Scope decisions:**
1. **One markdown file per component** — simpler than grouping (e.g. Pill+Tag share a file in code but get two docs).
2. **Concise template** — purpose line, Props table, 2 examples, 3–5 Do/Don't bullets. Aim for 40–80 lines per file. No long prose essays.
3. **No screenshots / storybook links** — not required by spec; text-only reference.
4. **Cross-link from top-level README** — add a one-line "## Design System" section pointing at `docs/design-system/README.md`.

**Out of scope:**
- Storybook / interactive playground
- Figma / design-token docs (already in the spec itself)
- Auto-generated prop tables (manually curated for now)
