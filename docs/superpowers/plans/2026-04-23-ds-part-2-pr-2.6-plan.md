# Design System SP1 Part 2 — PR 2.6 Implementation Plan (Hero + XP + Sparkline tokens)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship new `HeroBanner` primitive (gradient-only shell, no artwork), add `variant="xp"` to `ProgressBar` and adopt it on the 2 XP displays, and migrate `Sparkline`'s free-form `color` string prop to a tokenised `tone` union aligned with `ProgressBar`.

**Architecture:** HeroBanner is a full-bleed `<section>` with optional image + overlay + children slot. ProgressBar keeps its existing `tone` API for compatibility and gains a parallel `variant` prop (`default | xp`) — `xp` overrides tone to amber and adds a subtle box-shadow glow. Sparkline's `color?: string` is replaced with `tone` using the same 5-tone union ProgressBar uses (`primary | success | warn | danger | muted`); callsites pass dynamic direction mapped via a tiny helper.

**Tech Stack:** TS 5, React 19, Tailwind v4 (`@theme` CSS vars), Vitest 4 + RTL (jsdom pragma), `next/image`, `cn()`.

**Scope references:**
- Parent spec: `docs/superpowers/specs/2026-04-21-design-system-foundation-part-2-design.md` §3.10 (HeroBanner), §5 (Sparkline, ProgressBar), §10 PR 2.6
- Previous PR: 2.5 (merged — form primitives). Main tip: `83a3da9`.

**Deviations from parent spec (explicit so reviewer can push back):**
1. **Sparkline tone union** — spec §5 says `color="primary" | "accent"`. We mirror `ProgressBar`'s 5-tone union (`primary | success | warn | danger | muted`) instead. Rationale: the 2 existing callsites (`WeekMeasurementCard`, `SparklineCard`) map delta direction `good | bad | neutral` to 3 distinct colors (emerald / red / gray). A 2-value union (`primary | accent`) cannot express "bad" (red). ProgressBar's 5-tone set covers all current usage and stays API-consistent with ProgressBar. `accent` (amber) would alias `warn` token-wise so it's omitted to keep the union minimal.
2. **Sparkline prop name** — spec says prop is still called `color`. We rename to `tone` to match `ProgressBar`'s vocabulary. All callsites already migrate, so the rename costs nothing.
3. **ProgressBar xp shape** — spec §5 says "amber + subtle glow", no specific glow value. We use `shadow-[0_0_8px_rgba(245,158,11,0.4)]` (accent token at 40% alpha, 8px blur) on the inner fill. Small, non-distracting, matches existing `tier-glow` keyframe's amber family (`rgba(245,158,11,0.8)` at peak — we're softer since XP bar is always visible, not just on tier-up).
4. **HeroBanner adoption** — spec §10 says "ship HeroBanner shell" with no adoption line. Confirmed no existing callsites added in this PR; Dashboard adoption happens in SP3 alongside real artwork.

**Out of scope:**
- Markdown docs (PR 2.11)
- HeroBanner image artwork (SP3)
- ProgressBar `tone` token-var refactor (existing hex literals stay; not this PR's scope)
- Dashboard HeroBanner adoption (SP3)

```tsx
import { cn } from '@/components/ui/utils/cn'

type Tone = 'primary' | 'success' | 'warn' | 'danger' | 'muted'
type Variant = 'default' | 'xp'

type Props = {
  value: number | null
  max: number | null
  height?: number
  tone?: Tone
  variant?: Variant
  className?: string
}

const TONE: Record<Tone, string> = {
  primary: '#10b981',
  success: '#10b981',
  warn: '#f59e0b',
  danger: '#ef4444',
  muted: '#6b7280',
}

const XP_COLOR = '#f59e0b'
const XP_GLOW = 'shadow-[0_0_8px_rgba(245,158,11,0.4)]'

export function ProgressBar({
  value,
  max,
  height = 8,
  tone = 'primary',
  variant = 'default',
  className,
}: Props) {
  const pct = value != null && max != null && max > 0 ? Math.min((value / max) * 100, 100) : 0
  const fillColor = variant === 'xp' ? XP_COLOR : TONE[tone]
  return (
    <div
      className={cn('bg-border overflow-hidden rounded-full', className)}
      style={{ height }}
      role="progressbar"
      aria-valuenow={value ?? undefined}
      aria-valuemax={max ?? undefined}
    >
      <div
        className={cn(
          'h-full rounded-full transition-[width] duration-200',
          variant === 'xp' && XP_GLOW
        )}
        style={{ width: `${pct}%`, background: fillColor }}
      />
    </div>
  )
}
```

```tsx
import { sparklinePath } from '@/lib/sparkline'

type Tone = 'primary' | 'success' | 'warn' | 'danger' | 'muted'

type Props = {
  values: (number | null)[]
  width?: number
  height?: number
  tone?: Tone
  showEndDot?: boolean
  className?: string
}

const TONE: Record<Tone, string> = {
  primary: '#10b981',
  success: '#10b981',
  warn: '#f59e0b',
  danger: '#ef4444',
  muted: '#6b7280',
}

export function Sparkline({
  values,
  width = 120,
  height = 32,
  tone = 'muted',
  showEndDot = true,
  className,
}: Props) {
  const stroke = TONE[tone]
  const path = sparklinePath(values, width, height)
  if (!path) return <svg width={width} height={height} className={className} aria-hidden />
  let endIdx = -1
  for (let i = values.length - 1; i >= 0; i--) {
    if (values[i] != null) {
      endIdx = i
      break
    }
  }
  const stepX = values.length > 1 ? width / (values.length - 1) : 0
  const endX = endIdx * stepX
  const nums = values.filter((v): v is number => v != null)
  const min = Math.min(...nums)
  const max = Math.max(...nums)
  const range = max - min || 1
  const isConstant = max === min
  const endY = isConstant
    ? height / 2
    : height - (((values[endIdx] as number) - min) / range) * height
  return (
    <svg
      width={width}
      height={height}
      viewBox={`0 0 ${width} ${height}`}
      className={className}
      aria-hidden
    >
      <path
        d={path}
        fill="none"
        stroke={stroke}
        strokeWidth={2}
        strokeLinecap="round"
        strokeLinejoin="round"
      />
      {showEndDot && endIdx >= 0 && <circle cx={endX} cy={endY} r={3} fill={stroke} />}
    </svg>
  )
}
```

```tsx
import type { ReactNode } from 'react'
import Image from 'next/image'
import { cn } from '@/components/ui/utils/cn'

type Height = 'sm' | 'md' | 'lg'
type Overlay = 'dark' | 'gradient' | 'none'

type Props = {
  imageSrc?: string
  imageAlt?: string
  overlay?: Overlay
  height?: Height
  className?: string
  children: ReactNode
}

const HEIGHT_CLASS: Record<Height, string> = {
  sm: 'h-32',
  md: 'h-48',
  lg: 'h-64',
}

const BASE =
  'relative w-full overflow-hidden bg-gradient-to-br from-surface-raised via-surface to-background'

const OVERLAY_CLASS: Record<Exclude<Overlay, 'none'>, string> = {
  gradient: 'absolute inset-0 bg-gradient-to-t from-background/90 via-background/40 to-transparent',
  dark: 'absolute inset-0 bg-background/60',
}

export function HeroBanner({
  imageSrc,
  imageAlt = '',
  overlay = 'gradient',
  height = 'md',
  className,
  children,
}: Props) {
  return (
    <section role="region" className={cn(BASE, HEIGHT_CLASS[height], className)}>
      {imageSrc && (
        <Image
          src={imageSrc}
          alt={imageAlt}
          fill
          sizes="100vw"
          className="object-cover"
          priority
        />
      )}
      {overlay !== 'none' && (
        <div data-hero-overlay={overlay} className={OVERLAY_CLASS[overlay]} aria-hidden />
      )}
      <div className="relative z-10 flex h-full w-full items-end p-4">{children}</div>
    </section>
  )
}
```
