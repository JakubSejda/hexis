# Design System SP1 Part 2 — PR 2.4 Implementation Plan (Display Primitives)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship five display primitives — `Pill`, `Tag`, `Skeleton`, `EmptyState`, `Avatar` — each with full tests. Adoption deferred (see scope note below).

**Architecture:** Each primitive lives in `src/components/ui/primitive/`. Named exports, `cn()` for className, `React.forwardRef` on DOM-rooted primitives. Pill + Tag share a file (designed together, Tag extends Pill's styling surface). Skeleton, EmptyState, Avatar each get their own file. Lucide icons for EmptyState's `icon` prop and Tag's `onRemove` X.

**Tech Stack:** TypeScript 5, React 19, Tailwind CSS v4, Vitest 4 + RTL (jsdom per-file pragma), `lucide-react` (installed), `cn()` helper.

**Scope references:**
- Parent spec: `docs/superpowers/specs/2026-04-21-design-system-foundation-part-2-design.md` §3.4 (Pill/Tag), §3.5 (Skeleton), §3.6 (EmptyState), §3.9 (Avatar), §8 (testing), §10 PR 2.4
- Previous PR: PR 2.3 (merged as `6b1a73d` — Button/Card/Heading)

**Adoption scope — DEFERRED for this PR:**

Spec §10 PR 2.4 names three adoption targets: Pill on status badges, EmptyState on empty placeholders, Skeleton on dashboard tiles. A grep audit at plan time surfaced no unambiguous candidate sites in the current codebase (existing status-like elements are already inside Cards or domain components, not inline styled badges). Per spec's "scope confirmed during implementation" clause, PR 2.4 ships primitives only. Adoption will come opportunistically in subsequent PRs when natural sites appear (SP3 dashboard redesign for Skeleton; SP5 gamification for Pill status tags; SP5 empty screens for EmptyState).

**Name collision note — `Avatar`:**

`src/components/avatar/Avatar.tsx` already exists as a DOMAIN component (renders a tier-themed SVG: `/avatars/tier-${tier}.svg`). Its API is `{ tier, size?, className?, ringPulse? }` — completely different from the planned UI primitive `{ src?, alt, size?, fallback? }`. Both will coexist:
- Domain Avatar: `import { Avatar } from '@/components/avatar/Avatar'` — unchanged by this PR
- Primitive Avatar: `import { Avatar } from '@/components/ui'` (via top barrel → primitive barrel)

Consumers importing from both paths must alias one (`import { Avatar as TierAvatar } from '@/components/avatar/Avatar'`). No existing file imports both. Rename of the domain one to `TierAvatar` is a judgment call deferred to a future PR (likely when SP4 muscle visual language touches the tier artwork).

Create `src/components/ui/primitive/Pill.tsx`:

```tsx
import { forwardRef, type ReactNode, type HTMLAttributes, type MouseEvent } from 'react'
import { X } from 'lucide-react'
import { cn } from '@/components/ui/utils/cn'

type Variant = 'neutral' | 'success' | 'warning' | 'danger' | 'accent'
type Size = 'sm' | 'md'

const VARIANT_CLASS: Record<Variant, string> = {
  neutral: 'bg-surface-raised text-foreground',
  success: 'bg-primary-soft text-primary',
  warning: 'bg-accent-soft text-accent',
  danger: 'bg-danger text-background',
  accent: 'bg-accent text-background',
}

const SIZE_CLASS: Record<Size, string> = {
  sm: 'h-5 px-2 text-xs',
  md: 'h-6 px-3 text-sm',
}

const BASE = 'inline-flex items-center gap-1 rounded-md font-medium'

type PillProps = {
  variant?: Variant
  size?: Size
  className?: string
  children: ReactNode
} & Omit<HTMLAttributes<HTMLSpanElement>, 'className'>

export const Pill = forwardRef<HTMLSpanElement, PillProps>(function Pill(
  { variant = 'neutral', size = 'sm', className, children, ...rest }: PillProps,
  ref
) {
  return (
    <span
      ref={ref}
      className={cn(BASE, VARIANT_CLASS[variant], SIZE_CLASS[size], className)}
      {...rest}
    >
      {children}
    </span>
  )
})

type TagProps = {
  variant?: Variant
  size?: Size
  onRemove?: () => void
  onClick?: () => void
  className?: string
  children: ReactNode
}

export const Tag = forwardRef<HTMLSpanElement | HTMLButtonElement, TagProps>(function Tag(
  { variant = 'neutral', size = 'sm', onRemove, onClick, className, children }: TagProps,
  ref
) {
  const isInteractive = Boolean(onClick || onRemove)
  const classes = cn(BASE, VARIANT_CLASS[variant], SIZE_CLASS[size], className)

  if (!isInteractive) {
    return (
      <span ref={ref as React.Ref<HTMLSpanElement>} className={classes}>
        {children}
      </span>
    )
  }

  const handleRemoveClick = (e: MouseEvent<HTMLButtonElement>) => {
    e.stopPropagation()
    onRemove?.()
  }

  return (
    <button
      ref={ref as React.Ref<HTMLButtonElement>}
      type="button"
      className={classes}
      onClick={onClick}
    >
      {children}
      {onRemove ? (
        <span
          data-testid="tag-remove"
          role="button"
          tabIndex={-1}
          onClick={handleRemoveClick}
          className="ml-1 inline-flex cursor-pointer"
        >
          <X size={12} />
        </span>
      ) : null}
    </button>
  )
})
```

Key invariants:
- `Pill` is always a `<span>`.
- `Tag` is a `<span>` when neither `onClick` nor `onRemove` provided; `<button>` otherwise.
- `onRemove` click fires `onRemove`, calls `e.stopPropagation()`, does NOT fire `onClick`.
- Tag's X icon has `data-testid="tag-remove"` — test hook + click target.
- Both use `rounded-md` (token scale: 8px — smaller than button/card per spec §3.4).

Create `src/components/ui/primitive/Skeleton.tsx`:

```tsx
import type { CSSProperties, HTMLAttributes } from 'react'
import { cn } from '@/components/ui/utils/cn'

type Shape = 'text' | 'block' | 'avatar' | 'card'

const SHAPE_CLASS: Record<Shape, string> = {
  text: 'h-4 rounded',
  block: 'h-20 rounded-md',
  avatar: 'h-10 w-10 rounded-full',
  card: 'h-32 rounded-2xl',
}

type Props = {
  shape?: Shape
  lines?: number
  width?: string | number
  height?: string | number
  className?: string
} & Omit<HTMLAttributes<HTMLDivElement>, 'className'>

export function Skeleton({
  shape = 'text',
  lines = 1,
  width,
  height,
  className,
  ...rest
}: Props) {
  const base = 'animate-pulse bg-surface-raised'
  const widthClass = typeof width === 'string' ? width : undefined
  const style: CSSProperties = {
    ...(typeof width === 'number' ? { width: `${width}px` } : null),
    ...(typeof height === 'number' ? { height: `${height}px` } : null),
  }

  if (shape === 'text' && lines > 1) {
    return (
      <div className={cn('flex flex-col gap-2', className)} style={style} {...rest}>
        {Array.from({ length: lines }).map((_, i) => (
          <div
            key={i}
            data-testid="skeleton-line"
            className={cn(base, SHAPE_CLASS.text, widthClass)}
          />
        ))}
      </div>
    )
  }

  return (
    <div
      className={cn(base, SHAPE_CLASS[shape], widthClass, className)}
      style={style}
      {...rest}
    />
  )
}
```

Key invariants:
- Shape lookup table exhaustive over `Shape`.
- `lines > 1` with `shape="text"` renders a wrapper `<div>` + N child `<div>` (each with `data-testid="skeleton-line"`). The parent div gets forwarded props (data-testid, etc.).
- `lines=1` or any non-text shape renders a single `<div>` with the shape class.
- `width` as string is treated as a Tailwind class; as number, applied via inline `style.width`.
- `height` as number is applied via inline `style.height`; string case is out of scope (callers can use Tailwind via `className`).

```tsx
import type { ReactNode, HTMLAttributes } from 'react'
import type { LucideIcon } from 'lucide-react'
import { cn } from '@/components/ui/utils/cn'
import { Heading } from '@/components/ui/primitive/Heading'

type Props = {
  icon: LucideIcon
  title: string
  description?: string
  action?: ReactNode
  className?: string
} & Omit<HTMLAttributes<HTMLDivElement>, 'className'>

export function EmptyState({ icon: Icon, title, description, action, className, ...rest }: Props) {
  return (
    <div
      className={cn('flex flex-col items-center justify-center gap-3 py-10 text-center', className)}
      {...rest}
    >
      <Icon size={64} className="text-muted" />
      <Heading level={2}>{title}</Heading>
      {description ? <p className="text-muted max-w-sm text-sm">{description}</p> : null}
      {action ? <div className="pt-1">{action}</div> : null}
    </div>
  )
}
```

Key invariants:
- `icon` accepts a Lucide component reference (`LucideIcon` type), rendered at 64px with `text-muted`.
- Title uses the `Heading` primitive at `level={2}` — inherits semantic + visual styling.
- Description is a muted `<p>` with a max-width for readability.
- Action slot wraps any ReactNode (typically a `Button`).
- Centered layout via flex column + items-center.

Create `src/components/ui/primitive/Avatar.tsx`:

```tsx
import type { ReactNode, HTMLAttributes, ImgHTMLAttributes } from 'react'
import { cn } from '@/components/ui/utils/cn'

type Size = 'xs' | 'sm' | 'md' | 'lg' | 'xl'

const SIZE_CLASS: Record<Size, string> = {
  xs: 'h-6 w-6 text-xs',
  sm: 'h-8 w-8 text-xs',
  md: 'h-10 w-10 text-sm',
  lg: 'h-12 w-12 text-base',
  xl: 'h-16 w-16 text-lg',
}

const BASE = 'inline-flex items-center justify-center overflow-hidden rounded-full select-none'

type Props = {
  src?: string
  alt: string
  size?: Size
  fallback?: ReactNode
  className?: string
} & Omit<HTMLAttributes<HTMLElement>, 'className'>

function initialsFromAlt(alt: string): string {
  const parts = alt.trim().split(/\s+/).filter(Boolean)
  if (parts.length === 0) return ''
  if (parts.length === 1) return parts[0]!.charAt(0).toUpperCase()
  return (parts[0]!.charAt(0) + parts[parts.length - 1]!.charAt(0)).toUpperCase()
}

export function Avatar({ src, alt, size = 'md', fallback, className, ...rest }: Props) {
  const classes = cn(BASE, SIZE_CLASS[size], className)

  if (src) {
    const imgRest = rest as ImgHTMLAttributes<HTMLImageElement>
    return (
      // eslint-disable-next-line @next/next/no-img-element
      <img src={src} alt={alt} className={cn(classes, 'object-cover')} {...imgRest} />
    )
  }

  return (
    <div
      className={cn(classes, 'bg-surface-raised text-muted font-semibold')}
      aria-label={alt}
      role="img"
      {...(rest as HTMLAttributes<HTMLDivElement>)}
    >
      {fallback ?? initialsFromAlt(alt)}
    </div>
  )
}
```

Key invariants:
- With `src`: renders `<img>` with `alt`, rounded-full, object-cover for face-centering on non-square sources.
- Without `src`: renders `<div>` with initials derived from alt (first char of first + last word, uppercased), `role="img"` + `aria-label={alt}` for accessibility.
- `fallback` prop wins over initials.
- `alt` is REQUIRED — no default.
- No next/image: the spec explicitly doesn't call for it, and `<img>` suffices for user-profile avatars (raster sources; next/image benefits are marginal for 24–64px avatars). Includes `eslint-disable` for the `no-img-element` rule.
