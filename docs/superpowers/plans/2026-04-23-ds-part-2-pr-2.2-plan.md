# Design System SP1 Part 2 — PR 2.2 Implementation Plan (Folder Restructure)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the 8 existing root-level primitives into `src/components/ui/primitive/`, rename `LongPress.tsx` → `useLongPress.ts` (it's a hook), and normalize all consumer imports to go through the top barrel `@/components/ui`. **Zero behavioral change** — pure file move + mechanical import update.

The top barrel `src/components/ui/index.ts` hides the internal structure: every consumer imports from `@/components/ui`, never from nested paths. Spec calls this out explicitly in §7.

**Tech Stack:** TypeScript 5, Next.js 16 App Router, Vitest 4. No new dependencies. No new components. No new tests.

**Scope references:**
- Parent spec: `docs/superpowers/specs/2026-04-21-design-system-foundation-part-2-design.md` §7 (folder structure) + §10 PR 2.2 line item
- Previous PR: PR 2.1 (merged to main as `33b4999`) — added `cn()` helper, Container `size="full"`, Section `variant`

**Out of scope (do not do):**
- Refactor Stack/Grid/Divider to use `cn()` — those already live in `layout/` and don't need to move
- Migrate NumberInput to Input API — that's PR 2.5
- Convert ProgressBar/Sparkline variants — PR 2.6
- Any new primitives (Button/Card/Heading etc.) — PR 2.3+
- Docs markdown — PR 2.11
