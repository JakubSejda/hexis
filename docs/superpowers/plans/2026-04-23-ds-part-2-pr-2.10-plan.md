**Goal:** Mechanically replace all UI-chrome glyphs with Lucide icons across `src/**/*.tsx` AND satisfy spec §11.2 by normalizing the 28 remaining nested `@/components/ui/*/*` imports to either the top barrel `@/components/ui` (for consumers + tests) or relative paths (for library internals).

**Tech Stack:** `lucide-react` (ChevronRight, ArrowUp, ArrowDown, Check, AlertTriangle), no new dependencies.

- Parallel: no file overlap with #6/#7/#8/#9 expected — icon-chrome sites + nested-import debt are all in files those PRs don't touch, with two exceptions:
  - `StagnationBadge.tsx` — PR #7 swaps its native `title=` for Tooltip. This PR swaps its `⚠` glyph for `<AlertTriangle>`. If #7 merges first, rebase will need the AlertTriangle to live inside the Tooltip's child span; if this one merges first, #7 wraps the AlertTriangle in Tooltip. Either way 2-line conflict.
  - `XpBreakdown.tsx` — PR #9 swaps inline `<table>` for DataTable. This PR does NOT touch XpBreakdown (`3×` is multiplication, not icon). No conflict.

**Scope decisions (documented so reviewer can push back):**
1. **`×` multiplication sign stays as text** — 7 sites use `×` (U+00D7) as a multiplication operator in data display: `3×` (counts), `20kg × 5` (weight×reps), `Před×Po` (album name), plate calculator `20×2` etc. Spec §6 maps `×` to `<X size={16} />` but that's the close-icon semantic. Multiplication typography is not chrome; replacing it with a Lucide icon would destroy the numeric reading. Also covers `src/lib/1rm.ts` docstring and `src/db/bootstrap.ts` console.log (not UI at all).
2. **Internal `cn` imports → relative paths, not top barrel** — internal files should not import their own public barrel (creates a circular dependency risk + couples implementation to barrel shape). Use relative (`../utils/cn`, `./Heading`).
3. **Test files migrate to top barrel** — spec §11.2's grep is `src/`-wide and includes `src/tests/`. Tests use named exports that the barrel re-exports, so migration is behavior-preserving. Added benefit: tests now validate the public API surface (if a component stops being barrel-exported, its tests break — valid signal).
4. **Consumer pages `@/components/ui/layout` → `@/components/ui`** — stylistic consistency with the rest of the app (everyone else uses the top barrel).
5. **StagnationBadge `⚠` → `<AlertTriangle size={14} />`** — spec §6 doesn't enumerate `⚠` specifically but it's a semantic glyph and AlertTriangle is the Lucide equivalent.
6. **Final §11.2 gate:** `grep -rn "from '@/components/ui/[^']*/[^']*'" src/` must return 0 matches before push.

**Out of scope:**
- Replacing multiplication `×` anywhere
- Storybook / markdown docs (PR 2.11)
- Test strategy changes beyond the mechanical path swap

## Task 1: Icon sweep — replace UI-chrome glyphs with Lucide

**Files:** 8 files edited; 12 glyph sites swapped.

For every edit: keep surrounding text intact. Icons sit inside `inline-flex items-center gap-1` spans when adjacent to text. `ChevronRight` for ›; `ArrowUp`/`ArrowDown` for ↑/↓; `Check` for ✓; `AlertTriangle` for ⚠.

**Style convention:**
- When a glyph sits at end of text (e.g. `"Další ›"`), wrap parent in `inline-flex items-center gap-1` and append the icon.
- When a glyph prefixes text (e.g. `"↓ Načíst…"`, `"⚠ 3t. bez PR"`, `"✓ Zapsat sérii"`), prepend the icon inside the existing flex container.
- `aria-hidden` on decorative icons; actual semantics live on the parent button/link.
- Size: 12 for delta badges (tight row context), 14 for chrome chevrons and action affirms.
